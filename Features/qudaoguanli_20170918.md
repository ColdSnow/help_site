# 渠道管理（NEW）

通过此功能模块，您可进行网站或者APP的访问/下载来源以及后续深度用户行为跟踪。

**注：如您的iOS App想使用此新版的渠道管理功能，需在SDK集成时，在初始化代码中加入\[Growing setDeviceIDModeToIDFA\]此段代码，具体请参考：**[**iOS集成开发文档**](https://docs.growingio.com/SDK/iOS.html)**。**

## 网站/WAP-渠道管理使用说明

对于网站/H5产品，GIO通过 UTM 参数区分不同链接来源，进行访问，跳出率等行为统计。

### 使用方法

1. 点击进入GIO后台，渠道管理，选择您需要推广的网站。
2. 进入“配置与管理”模块“推广管理”，点击“新建监测”。
3. 按照系统要求填写内容。监测链接为您自定义命名，示例：百度618投放。推广活动也是您自定义的维度，如：市场部八月投放等。
4. 点击保存。
5. 获取监测链接，配置到投放后台。![](/assets/qudaoguanli_20170918/1.png)

### 数据报表

1. 点击选择您要查看数据的网站，进入“基础数据” ，“推广详细”模块。
2. 推广数据可分天查看，也可查看今天分小时数据。今天数据统计有1-2小时延迟。
3. 有小箭头的字段支持排序。
4. 有筛选图标的字段支持筛选，支持的字段有：目标渠道，监测链接，推广活动。
5. 查询结果可直接下载。![](/assets/qudaoguanli_20170918/2.png)

### 指标说明

1. 访问用户量：访问用户量之和。
2. 访问次数：访问次数。
3. 新访问用户数：过去365天首次访问网站的用户。
4. 跳出率：所有会话（session）中，计算只有一个访问页面的会话占全部会话的比例。 用于衡量落地页质量。
5. 平均访问时长：平均每次访问\(会话\)在网站上的停留时间。访问时长=访问时长/访问次数。体现对访客的吸引程度。
6. 平均访问页数：平均每次访问在网站上访问的页面数。体现对访客的吸引程度。
7. 点击：链接点击次数之和。

### 配置与管理

1. 点击进入推广管理模块，查看所有推广监测链接。
2. 点击“新建监测”，进行链接新建。
3. 点击“监测名称”，弹出更多信息，包括创建人，更新时间等。
4. 最右侧操作栏，点击编辑按钮可编辑名称及跳转地址。![](/assets/qudaoguanli_20170918/3.png)
5. 点击进入“渠道管理”模块进行渠道管理以及新建渠道。原则上，当系统有公有渠道时，不建议再新增同名渠道。
6. 对渠道名称，类型，创建者支持筛选。
7. 点击右侧“编辑”按钮进行名称编辑。（仅限自定义渠道）![](/assets/qudaoguanli_20170918/4.png)

### 统计指标说明

1. 点击：即每条监测链接统计到的点击次数。
2. 激活：下载APP后首次联网打开定义为一个激活。
3. 转化率：激活除以点击，百分比显示。
4. 新登录：当用户属性CS1字段被填写入一个之前不存在的值，即视为新登录。详情参考GIO SDK加载模块中“用户自定义维度”相关说明。
5. 次日留存：用户激活后，第二天再次打开APP，产生1个以上PV，视为产生一次次日留存。


### 归因逻辑

1. 如何定义用户？用户唯一设备号？
 对于安卓应用，GIO优先使用IMEI号进行精准激活匹配，没有IMEI的情况下采用AndroidID匹配，如果也没有获取到AndroidID，则采用IP+UA的方式模糊匹配。 对于iOS应用，GIO优先使用IDFA进行精准激活匹配，没有IDFA则使用IP+UA的方式模糊匹配。 
2. 归因规则是？
 按照Last Click（最近点击）规则，同时辅以反作弊规则进行归因。系统归因时间窗默认15天内，即一个激活存在被归因到15天前的点击的情况。

### 渠道配置指南

#### 核心原则

若生成监测链接的过程您填写过目标跳转地址，则GIO生成的链接复制后填写到投放后台诸如“点击地址”“Landing跳转地址”“目标地址”这样的内容上，若生成监测链接的过程没有填写目标跳转地址，则填写到投放平台“第三方点击监测地址”“点击监测地址”这样的内容字段上。

#### 今日头条

事先准备： 进入GIO后台，为今日头条的投放生成一条监测链接。进入“渠道配置”模块，点击“今日头条”，“是否回传激活”字段设置为“是”。![](/assets/qudaoguanli_20170918/9.png)

1. 进入头条后台，“工具箱”。

2. 点击“转化跟踪”。

3. 进行移动应用APP API转化新建。输入应用下载地址，比如APP的iTunes下载地址。

4. 填写GIO后台生成的监测链接。根据您SDK的版本，生成的链接可能有两种形式，如果您的SDK版本低于1.0.3，那么您将生成以下列形式开始链接：[https://t.growingio.com/app/at0/](https://t.growingio.com/app/at0/) ；如果您的SDK版本等于或高于1.0.3，那么您将以下列形式开始的链接：[https://t.growingio.com/app/at1/](https://t.growingio.com/app/at1/) 。您会发现，外观上他们区别主要在于末尾的at0与at1。但从业务上来看，两类链接支持不同的统计方式。低于1.0.3版本的用户所生成的链接无法支持用设备号精确匹配下载激活转化。

   对于大于1.0.3版本的用户，也就是[https://t.growingio.com/app/at1/](https://t.growingio.com/app/at1/)此类链接的用户，请您继续参考本流程继续完成配置。对于小于或等于1.0.3版本的用户，请您忽略以下步骤，直接将取得的链接放置于创意“点击地址”之类的内容字段上。同时，也建议您尽快将GIO SDK版本提升到1.0.3以上。

5. 配置完成后，请按照今日头条自动联调流程完成后续过程，联调成功后进行计划引用。 截图示意：![](/assets/qudaoguanli_20170918/10.png)

### 2.投放的渠道没有对接怎么办？  

 联系您在GIO的客户成功或者发送邮件到zhouxiuqiong@growing.io,GIO三日内评估是否能对接。如果不能对接，您可自定义这个渠道，使用GIO短链，系统会使用IP+UA的方式模糊匹配。

### 3.GIO的广告监测产品与其他监测产品有何差异？

 GIO使用无埋点方案搜集用户行为数据，除了标准的激活，登录，留存事件系统默认统计外，您可进行指标自定义，与广告监测链接进行关联，深度分析广告效果。

### 5.可以导出原始日志吗？  

  可以，详见：[https://docs.growingio.com/original\_data\_export.html](https://docs.growingio.com/original_data_export.html)。

### 6.与单图部分怎么结合？

由于整个移动应用部分优化了归因模型，以前使用UTM的维度请按照以前的广告来源，广告内容等UTM维度查看。新监测请引用“移动应用-目标渠道”“移动应用-推广互动”“移动应用-监测链接”三个维度。目前显示的为ID，即将优化为显示名称。![](/assets/qudaoguanli_20170918/16.png)

### 7.转化联调不成功怎么办？ 

  1. 请确保是否正确加载SDK1.0.3以上。与今日头条和广点通等广告平台的联调依赖SDK获取IDFA&IMEI，若获取不到则加载失败。  
  2. 请检查在广告平台的填写内容是否与此文档说明一致。请不要将广点通渠道的链接配置到今日头条。  
  3. 请确保用于联调的手机之前没有安装过您的推广应用。  
  4. 请确保您接入GIO SDK进行数据采集时没有采样。  
  5. 以上若都确认仍无结果，请联系相关客户经理，技术支持或者邮件至adtrack@growing.io

### 8.我可以同时生成一个链接既监测iOS APP下载效果又监测安卓APP下载效果吗？ 

  不可以。请确保一个监测对应一个APP（分操作系统）。

### 9.激活概览模块的激活与推广详细模块的激活为什么对不上？

  激活概览模块，以激活时间戳统计激活。推广详细模块，以点击时间戳统计激活。举例： 9.18日用户点击广告，9.20日激活APP。则此激活在激活概览模块出现在9.20，但是在激活详细模块，呈现为9.18的某个点击带来一个激活。

### 10.渠道管理模块的新登录与单图里的新登录为什么有差异？

  1. 渠道管理模块的新登录，会进行设备去重。其定义为激活的设备中有多少设备产生了新登录。而单图中未做设备去重，其定义为选定的时间内有多少新登录账号产生。  
  2. 双方统计时间戳不一致。单图中为选定时间内有多少新登录，渠道管理模块为选定的时间内的激活人群或者点击人群有多少后续产生过新登录

### 11.如何监测不同广告计划，素材的效果，需要分配多个监测链接吗？ 

  对于支持激活回传的渠道\(如今日头条，广点通等\)，您只需要生成一条监测链接并新建转化联调通过即可。通过后，将您的推广计划与转化绑定，每个计划，单元，创意的转化效果均可以在广告平台后台可查看。
